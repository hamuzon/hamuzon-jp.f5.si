β 4.0 (ベータ版)

JSONファイル名の命名規則の違い

v1.0 のファイル名

calendar_events_[YYYY][MM][DD]_[HH][mm][ss].json

例: calendar_events_20250710_183015.json

v2.0 のファイル名

Calendar-[CALENDAR_VERSION]_[YYYY]_[MM]_[DD]_[HH]_[mm]_[ss].json

例: Calendar-2.0_2025_07_10_18_30_15.json

v3.0 のファイル名

Calendar-[CURRENT_SAVE_VERSION]_[YYYY]_[MM]_[DD]_[HH]_[mm]_[ss].json

例: Calendar-3.0_2025_07_10_18_30_15.json

v1.0 と v2.0（v3.0）はファイル名やJSON内容が異なります。
ただし、v3.0 は v2.0 のJSON形式と互換性があります。 

β 4.0 は一部改良を予定しており、v1.0～v4.0 のJSON形式に対応しています。
ファイル名は以下の形式です。
Calendar-[CURRENT_SAVE_VERSION]_[YYYY]-[MM]=[DD]_[HH]_[mm]_[ss].json
例: Calendar-4.0_2025-07-10_18_30_15.json
⚠️ 注意: v1.0 と v2.0を同時に使用すると、片方のversionのレンダー表示が崩れることがあります。
一度に1つのバージョンのみの利用を推奨します。

Differences in JSON Filename Format

v1.0 format

calendar_events_[YYYY][MM][DD]_[HH][mm][ss].json

Example: calendar_events_20250710_183015.json

v2.0 format

Calendar-[CALENDAR_VERSION]_[YYYY]_[MM]_[DD]_[HH]_[mm]_[ss].json

Example: Calendar-2.0_2025_07_10_18_30_15.json

v3.0 format

Calendar-[CURRENT_SAVE_VERSION]_[YYYY]_[MM]_[DD]_[HH]_[mm]_[ss].json

Example: Calendar-3.0_2025_07_10_18_30_15.json

Filenames and JSON contents differ between v1.0 and v2.0/v3.0.
However, v3.0 is backward compatible with the JSON format of v2.0. 

Beta version 4.0 includes some improvements and supports JSON formats from v1.0 to v4.0.
Filename format is as follows:
Calendar-[CURRENT_SAVE_VERSION]_[YYYY]-[MM]=[DD]_[HH]_[mm]_[ss].json
Example	: Calendar-4.0_2025-07-10_18_30_15.json
⚠️ Caution: Using v1.0 and v2.0 at the same time may result in distorted rendering of one of the versions.
We recommend using only one version at a time.
